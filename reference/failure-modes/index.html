    Failure Modes¶

    Generated from the strict class taxonomy.

    Agent Cognition & Orchestration¶

    Trace-observable failures in intent formation, planning, control flow, delegation, and termination. This aligns to Dimension I in Shah et al. (2026).

    choice_overload_premature_commitment¶

    Choice Overload (Premature Commitment)

    Under a large branching factor, the agent commits too early to one plan, interpretation, or tool without sufficient exploration or backtracking.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range medium -> high

    Typical triggers¶

    • Ambiguous goals
    • High-dimensional action spaces
    • Long-horizon tool-rich tasks
    • Budgeted reasoning that forces early stopping
    • Missing explore-versus-exploit logic

    Detection signals¶

    • Low plan diversity in reasoning traces
    • Early tool lock-in despite ambiguous inputs
    • Frequent late replans after initial commitment
    • Repeated failure on tasks with large branching factors
    • Low sensitivity to new evidence after an initial choice

    Mitigations¶

    • Use deliberative search with checkpoints and backtracking
    • Add reject or ask-human branches for uncertain tool selection
    • Allocate an explicit exploration budget before irreversible steps
    • Stress test tasks with multiple plausible solution paths

    Related failure modes¶

    • goal_intent_mismatch
    • insufficient_planning_long_horizon_brittleness
    • premature_termination

    goal_intent_mismatch¶

    Goal / Intent Mismatch

    The agent forms an internal objective that differs from the user's real intent, producing locally coherent actions that are globally wrong.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range medium -> critical

    Typical triggers¶

    • Vague or shorthand prompts
    • Hidden preferences or stakeholder conflicts
    • Missing domain constraints
    • Context omissions during long sessions
    • Weak clarification-seeking behavior

    Detection signals¶

    • Low clarification rate before execution
    • High rate of user corrections or undo requests
    • Semantic mismatch between extracted requirements and later actions
    • Plans that optimize a proxy objective instead of the stated goal
    • Repeated comments that the agent is solving the wrong problem

    Mitigations¶

    • Use structured intent elicitation before execution
    • Fail closed when intent remains underspecified
    • Capture explicit preference and constraint schemas
    • Require confirmation before irreversible steps

    Related failure modes¶

    • choice_overload_premature_commitment
    • constraint_omission_policy_noncompliance
    • reasoning_action_mismatch

    insufficient_planning_long_horizon_brittleness¶

    Insufficient Planning / Long-Horizon Brittleness

    The agent produces incomplete or fragile plans that fail under multi-step dependencies, dynamic environments, or long context histories.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range medium -> high

    Typical triggers¶

    • Long-horizon tasks
    • Dynamic or partially observed environments
    • Multi-tool workflows
    • Long context histories
    • Weak world models or state tracking

    Detection signals¶

    • High replanning frequency
    • Increasing plan-versus-execution divergence over time
    • Rising stuck rates on deeper dependency chains
    • Missing recovery after failed steps
    • Repeated inability to represent preconditions or effects

    Mitigations¶

    • Use structured planning representations and checkpoints
    • Integrate failure recovery into the control loop
    • Benchmark on long-horizon tasks rather than only short tasks
    • Restrict autonomy on irreversible long-horizon operations

    Related failure modes¶

    • choice_overload_premature_commitment
    • premature_termination
    • reasoning_action_mismatch

    premature_termination¶

    Premature Termination

    The agent declares success or stops executing before the objective has been verified or completed.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range low -> high

    Typical triggers¶

    • Budget constraints
    • Weak termination criteria
    • Tool errors misread as success
    • Missing progress metrics
    • Lack of validator-gated completion

    Detection signals¶

    • Final done-like messages without a preceding verification step
    • Stopping after partial execution despite unresolved work
    • Success claims with no validator or test output
    • Missing state deltas that confirm completion
    • Traces that end immediately after an error without recovery

    Mitigations¶

    • Gate completion on validators, tests, or explicit confirmation
    • Require evidence of completion in the trace
    • Add safe-abort and escalation paths instead of silent early exit
    • Track progress metrics independently of assistant text

    Related failure modes¶

    • verification_failure
    • step_repetition_loop
    • insufficient_planning_long_horizon_brittleness

    Failure-specific attributes¶

    Attribute Range Description
    validation_surface string Validator, test surface, or explicit completion check expected before termination.

    step_repetition_loop¶

    Step Repetition Loop

    The agent repeats identical or cyclic actions without making progress, wasting resources and potentially repeating side effects.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range low -> high

    Typical triggers¶

    • Inadequate loop detection
    • Missing environment state deltas
    • Weak progress metrics
    • Tool failures followed by naive retries
    • Miscalibrated utility or reward heuristics

    Detection signals¶

    • Repeated identical tool calls with the same arguments
    • Cyclic action sequences
    • Unchanged environment outputs across retries
    • Rising cost with no new evidence
    • Repeated attempts after a deterministic schema failure

    Mitigations¶

    • Add loop detectors and repeated-action caps
    • Compare tool results before retrying
    • Escalate after bounded retries instead of continuing blindly
    • Use validator-gated state changes to confirm progress

    Related failure modes¶

    • premature_termination
    • tool_misuse
    • resource_exhaustion_agent_denial_of_service

    Failure-specific attributes¶

    Attribute Range Description
    repeated_tool_signature string Tool signature repeatedly invoked without meaningful state change.
    repeated_call_count integer Number of repeated identical calls observed in the loop.

    reasoning_action_mismatch¶

    Reasoning-Action Mismatch

    The agent's stated plan or reasoning does not match the actions it actually executes, degrading trust and increasing hazard.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range medium -> high

    Typical triggers¶

    • Tool-call formatting errors
    • Latent instruction overrides
    • Hallucinated tool outputs
    • UI or telemetry gaps between plan and execution
    • Weak grounding between text plans and tool schemas

    Detection signals¶

    • Semantic divergence between reasoning text and tool calls
    • Audit trail discrepancies
    • Claimed validations that do not appear in the trace
    • Actions that contradict immediately preceding intent statements
    • Tool outputs referenced before they exist

    Mitigations¶

    • Use typed tool interfaces and constrained decoding
    • Prefer machine-checkable executable plans
    • Log arguments and outputs for every action
    • Differentially test reasoning traces across seeds

    Related failure modes¶

    • goal_intent_mismatch
    • tool_misuse
    • verification_failure

    multi_agent_coordination_breakdown¶

    Multi-Agent Coordination Breakdown

    Multiple agents fail to coordinate because of role violations, ignored inputs, withheld information, or shared-state failures.

    Field Value
    Category Agent Cognition & Orchestration
    Severity range medium -> critical

    Typical triggers¶

    • Ambiguous role definitions
    • Conflicting incentives
    • Missing shared-memory protocols
    • Long multi-agent dialogues
    • Weak arbitration or consensus logic

    Detection signals¶

    • Inter-agent contradictions
    • Ignored inputs or missing handoffs
    • Conversation resets in multi-agent flows
    • Divergence in agent beliefs over time
    • Consensus failure counters or deadlocks

    Mitigations¶

    • Use explicit protocols for roles, turn-taking, and arbitration
    • Version shared state across agents
    • Assign a supervisor with limited authority and validation duties
    • Evaluate multi-agent traces separately from single-agent traces

    Related failure modes¶

    • state_desynchronization_context_loss
    • observability_provenance_gaps
    • reasoning_action_mismatch

    Tooling, Integration & Actuation¶

    Failures at the boundary between agent reasoning and deterministic tools, APIs, permissions, and external systems. This aligns to Dimension II in Shah et al. (2026).

    constraint_omission_policy_noncompliance¶

    Constraint Omission / Policy Noncompliance

    The agent fails to model or enforce explicit constraints such as tool permissions, safety policies, or regulatory requirements.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range high -> critical

    Alignment Metadata¶

    Key Value
    alignment_status trace_adaptation
    cross_cutting_concerns policy_compliance
    security

    Typical triggers¶

    • Vaguely specified tools
    • Overlapping tool capabilities
    • Missing approval or permission checks
    • Policy encoded only as prompt text
    • Weak separation between instructions and untrusted content

    Detection signals¶

    • Authorization denials in tool results
    • Policy-violation markers
    • Capability overreach relative to the stated task
    • Anomalous resource access patterns
    • Approval-required actions attempted without confirmation

    Mitigations¶

    • Use least-privilege tool scopes and capability-based APIs
    • Enforce executable policy checks before action
    • Add sandboxing around file and network mutation
    • Require auditable chains of events for high-risk domains

    Related failure modes¶

    • unauthorized_actuation_permission_boundary_violation
    • prompt_injection_indirect_instruction_hijacking
    • tool_misuse

    wrong_tool_selection¶

    Wrong Tool Selection

    The agent chooses an inappropriate tool for the task, even if the chosen tool is invoked correctly.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range medium -> critical

    Typical triggers¶

    • Overlapping tool capabilities
    • Vague tool descriptions
    • Weak tool-choice training
    • Partial observability about required parameters
    • Missing reject-or-defer option

    Detection signals¶

    • Tool failures that would not occur with a more appropriate tool
    • Endpoint mismatch relative to stated intent
    • Detours through unrelated tools before success
    • High retry volume across several incompatible tools
    • Explicit comments that the wrong tool was used

    Mitigations¶

    • Provide clearer tool contracts and examples
    • Add reject and ask-human branches for uncertain cases
    • Benchmark selection accuracy under noisy conditions
    • Restrict sensitive tools behind additional policy checks

    Related failure modes¶

    • tool_misuse
    • reasoning_action_mismatch
    • constraint_omission_policy_noncompliance

    tool_misuse¶

    Tool Misuse

    The agent selects a plausible tool but invokes it with invalid arguments, wrong sequencing, or schema mismatches that cascade into failure.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range medium -> critical

    Typical triggers¶

    • Contract mismatches between generation and API schemas
    • Validation or type errors
    • Partial observability over required fields
    • Ambiguous tool semantics
    • Retry behavior that replays the same invalid call

    Detection signals¶

    • Schema validation errors
    • Missing required argument errors
    • JSON decode failures in tool pipelines
    • Repeated identical tool retries after deterministic failures
    • Unknown or out-of-range fields in tool payloads

    Mitigations¶

    • Enforce strict schemas and static checks
    • Simulate tools during evaluation
    • Short-circuit repeated invalid retries
    • Require ask-human or reject flows for uncertain arguments

    Related failure modes¶

    • wrong_tool_selection
    • step_repetition_loop
    • unsafe_tool_output_handling

    Failure-specific attributes¶

    Attribute Range Description
    tool_name string Tool implicated in the misuse event.
    interface_error_kind string Tool-interface or invocation error kind observed in the trace.

    unsafe_tool_output_handling¶

    Unsafe Tool-Output Handling

    Tool outputs are treated as trusted instructions or executable content without sanitization or provenance checks.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range high -> critical

    Typical triggers¶

    • LLM-output-to-executor pipelines
    • Retrieval-augmented browsing
    • Code-writing or file-writing agents
    • Automated email or workflow execution
    • Missing trust boundaries around external content

    Detection signals¶

    • Executable payloads flowing directly into tools
    • Anomalous file writes or network calls after retrieval
    • Injection signatures in tool outputs
    • Missing provenance for content later treated as instructions
    • Security scanner alerts after rendering or execution

    Mitigations¶

    • Sanitize and escape untrusted outputs before execution
    • Enforce data-versus-instruction boundaries
    • Sandbox code execution and file mutation
    • Red-team output-to-action workflows

    Related failure modes¶

    • prompt_injection_indirect_instruction_hijacking
    • tool_misuse
    • unauthorized_actuation_permission_boundary_violation

    unauthorized_actuation_permission_boundary_violation¶

    Unauthorized Actuation / Permission Boundary Violation

    The agent performs actions beyond its authorized scope, often because action gating or identity checks are weak.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range critical -> critical

    Typical triggers¶

    • Overbroad credentials
    • Shared service accounts
    • Missing confirmation for irreversible actions
    • Weak auth integration
    • Instruction hijacking that bypasses approval intent

    Detection signals¶

    • Authorization denials followed by retries
    • Privilege-escalation patterns
    • Resource access outside task scope
    • Unapproved mutation of sensitive systems
    • High-entropy roaming across tools and resources

    Mitigations¶

    • Enforce per-action authorization
    • Use escrowed credentials and segmented environments
    • Require confirmation for irreversible actions
    • Audit and review all sensitive-tool access

    Related failure modes¶

    • constraint_omission_policy_noncompliance
    • prompt_injection_indirect_instruction_hijacking
    • agent_compromise_instruction_hijack

    agent_compromise_instruction_hijack¶

    Agent Compromise / Instruction Hijack

    The agent's control flow is hijacked by malicious instructions or compromised intermediaries so that it acts on attacker objectives.

    Field Value
    Category Tooling, Integration & Actuation
    Severity range high -> critical

    Alignment Metadata¶

    Key Value
    alignment_status trace_extension
    cross_cutting_concerns security
    adversarial

    Typical triggers¶

    • Prompt injection reaching high-privilege contexts
    • Compromised plugins, tools, or connectors
    • Overbroad credentials
    • Missing integrity checks on intermediate messages
    • Unsafe delegation chains

    Detection signals¶

    • Abrupt changes in objective after untrusted content ingestion
    • New instructions that bypass normal approvals
    • Actions inconsistent with prior policy constraints
    • Sensitive tool use immediately after suspicious input
    • Compromise indicators in intermediary tool outputs

    Mitigations¶

    • Harden trust boundaries around plugins and retrieved content
    • Require policy checks before privileged actions
    • Limit delegation authority and credential scope
    • Audit integrity of intermediaries and connectors

    Related failure modes¶

    • prompt_injection_indirect_instruction_hijacking
    • unauthorized_actuation_permission_boundary_violation
    • targeted_knowledge_base_poisoning

    Perception, Context & Memory¶

    Failures rooted in input interpretation, factual grounding, state tracking, and persistent memory. This aligns to Dimension III in Shah et al. (2026).

    hallucination_fabricated_evidence¶

    Hallucination / Fabricated Evidence

    The agent generates incorrect facts, invented citations, or fabricated evidence that then contaminates downstream reasoning or action.

    Field Value
    Category Perception, Context & Memory
    Severity range medium -> critical

    Typical triggers¶

    • Ungrounded open-domain queries
    • Domain-specific nomenclature gaps
    • Long-horizon summarization
    • Citation generation without provenance
    • Weak retrieval alignment

    Detection signals¶

    • Fact-checker disagreement
    • Invalid citations or missing provenance
    • Claims unsupported by any retrieved source
    • Contradictions with authoritative databases
    • Reasoning that references nonexistent outputs

    Mitigations¶

    • Require claim-level provenance
    • Use retrieval-backed verification
    • Add abstention behavior under uncertainty
    • Restrict autonomous action when factual certainty is low

    Related failure modes¶

    • verification_failure
    • reasoning_action_mismatch
    • observability_provenance_gaps

    memory_poisoning_persistent_instruction_corruption¶

    Memory Poisoning / Persistent Instruction Corruption

    Malicious or contaminated content is written into persistent memory so future recalls manipulate later behavior.

    Field Value
    Category Perception, Context & Memory
    Severity range critical -> critical

    Alignment Metadata¶

    Key Value
    alignment_status trace_extension
    cross_cutting_concerns security
    adversarial

    Typical triggers¶

    • Autonomous memorization
    • Shared memory across agents
    • Unvalidated writes into RAG or memory stores
    • Treating recalled memory as trusted
    • Missing authorization around memory mutation

    Detection signals¶

    • Untrusted memory-write events
    • Behavior drift correlated with recalled items
    • Retrievals that reintroduce malicious instructions
    • Missing provenance on memory content
    • Persistent recurrence across sessions

    Mitigations¶

    • Authenticate and validate memory writes
    • Separate trusted and untrusted memory tiers
    • Expose memory inspection and remediation to operators
    • Continuously monitor memory mutations

    Related failure modes¶

    • targeted_knowledge_base_poisoning
    • prompt_injection_indirect_instruction_hijacking
    • state_desynchronization_context_loss

    prompt_injection_indirect_instruction_hijacking¶

    Prompt Injection / Indirect Instruction Hijacking

    Adversarial instructions in user input or retrieved third-party content cause the agent to ignore its original goals and controls.

    Field Value
    Category Perception, Context & Memory
    Severity range high -> critical

    Alignment Metadata¶

    Key Value
    alignment_status trace_extension
    cross_cutting_concerns security
    adversarial

    Typical triggers¶

    • Browsing or retrieval-augmented generation
    • Email or knowledge ingestion
    • Poor separation between data and instructions
    • Untrusted pages or documents reaching high-privilege tools
    • Weak allowlisting around tool calls

    Detection signals¶

    • Sudden goal drift after retrieval
    • Instruction override cues in untrusted content
    • Suspicious tool-call sequences after fetch operations
    • Monitor alerts for prompt-injection signatures
    • New directives that conflict with prior constraints

    Mitigations¶

    • Isolate untrusted content from tool execution
    • Enforce deterministic policy checks and allowlists
    • Add robust sandboxing and sanitization
    • Continuously adversarially test retrieval chains

    Related failure modes¶

    • unsafe_tool_output_handling
    • agent_compromise_instruction_hijack
    • unauthorized_actuation_permission_boundary_violation

    state_desynchronization_context_loss¶

    State Desynchronization / Context Loss

    Conversation history, task state, or environment assumptions diverge from reality, leading to locally coherent but globally incorrect behavior.

    Field Value
    Category Perception, Context & Memory
    Severity range medium -> high

    Typical triggers¶

    • Long sessions
    • Context compaction or truncation
    • Tool outputs not incorporated into state
    • Concurrency or conversation resets
    • Weak serialization of intermediate state

    Detection signals¶

    • Explicit compaction or reset markers in the trace
    • Replay divergence after a context boundary
    • Sudden amnesia or loss-of-history messages
    • Corrective loops after dropped context
    • Inconsistencies between stored state and new observations

    Mitigations¶

    • Use explicit state machines and snapshots
    • Version and compress context with invariants
    • Replay traces during debugging
    • Inject context truncation faults during testing

    Related failure modes¶

    • multi_agent_coordination_breakdown
    • memory_poisoning_persistent_instruction_corruption
    • observability_provenance_gaps

    Failure-specific attributes¶

    Attribute Range Description
    state_reset_marker string Explicit marker of conversation reset, context compaction, or state loss.

    targeted_knowledge_base_poisoning¶

    Targeted Knowledge-Base Poisoning

    External knowledge stores, retrieval corpora, or indexed artifacts are deliberately poisoned so the agent later reasons from corrupted evidence.

    Field Value
    Category Perception, Context & Memory
    Severity range high -> critical

    Alignment Metadata¶

    Key Value
    alignment_status trace_extension
    cross_cutting_concerns security
    adversarial

    Typical triggers¶

    • Unvalidated retrieval corpora
    • Shared indexes fed by external contributors
    • Weak provenance on imported documents
    • Autonomous ingestion of third-party content
    • Missing trust-tier separation in knowledge stores

    Detection signals¶

    • Drift tied to specific retrieved documents
    • Recurrent malicious or low-quality sources in evidence chains
    • Missing provenance or trust level on retrieved artifacts
    • Retrieval results that consistently redirect goals
    • Behavior changes after index refreshes or imports

    Mitigations¶

    • Validate and sign knowledge-base updates
    • Track provenance and trust tiers for retrieved content
    • Quarantine untrusted corpora from privileged workflows
    • Continuously red-team retrieval and ingestion channels

    Related failure modes¶

    • memory_poisoning_persistent_instruction_corruption
    • prompt_injection_indirect_instruction_hijacking
    • hallucination_fabricated_evidence

    Runtime & Environment Grounding¶

    Failures rooted in runtime dependencies, platform assumptions, resource limits, and execution-environment coupling. This aligns to Dimension IV in Shah et al. (2026).

    resource_exhaustion_agent_denial_of_service¶

    Resource Exhaustion / Agent Denial of Service

    The agent exhausts tokens, compute, time, or tool quotas, causing service disruption or unsafe partial execution.

    Field Value
    Category Runtime & Environment Grounding
    Severity range medium -> high

    Alignment Metadata¶

    Key Value
    alignment_status trace_adaptation

    Typical triggers¶

    • Unbounded loops
    • Large tool outputs
    • Adversarial cost-maximizing prompts
    • Missing rate limits or circuit breakers
    • Weak caching and retrieval bounds

    Detection signals¶

    • Token budget utilization near the model context window
    • Quota exhaustion or insufficient-quota errors
    • Latency spikes with no corresponding progress
    • Rising cost across repeated failures
    • Explicit token limit or rate limit markers

    Mitigations¶

    • Enforce strict budgets and circuit breakers
    • Use progressive summarization
    • Bound retrieval size and retry volume
    • Add adversarial cost testing before deployment

    weak_fallback_silent_degradation¶

    Weak Fallback (Silent Degradation)

    The agent silently enters a degraded operating mode such as a fallback model, reduced toolset, cached retrieval path, or heuristic shortcut without surfacing that downgrade to the operator.

    Field Value
    Category System Reliability & Observability
    Severity range medium -> critical

    Typical triggers¶

    • Tool or API failures
    • Rate limits or partial outages
    • Token or compute constraints
    • Missing permissions on the preferred path
    • Uninstrumented graceful-degradation logic

    Detection signals¶

    • Sudden changes in model identifiers across adjacent events
    • Tool-call volume or success rate drops
    • Spikes in best-effort or fallback phrasing
    • Divergence between offline evaluation quality and online trace behavior
    • Increased run-to-run variance after a failure

    Mitigations¶

    • Emit explicit degraded-mode flags in telemetry and user-visible output
    • Fail closed for high-stakes operations rather than silently degrading
    • Evaluate fallback paths independently from primary paths
    • Maintain incident playbooks for partial outages and fallback activation

    verification_failure¶

    Verification Failure

    The agent performs no verification, incomplete verification, or incorrect verification, allowing bad outputs to pass as successful.

    Field Value
    Category System Reliability & Observability
    Severity range high -> high

    Typical triggers¶

    • Missing validators
    • Same-model actor and judge
    • Weak tests or no executable checks
    • Missing ground truth
    • Mis-specified evaluation metrics

    Detection signals¶

    • Success claims that lack supporting evidence
    • High false-pass rates
    • Disagreement among independent verifiers
    • Validator coverage gaps
    • Internal success markers that contradict external evidence

    Mitigations¶

    • Use independent validators and executable checks
    • Diversify verification tools or models
    • Evaluate the monitors themselves
    • Retain verification evidence in trace logs

    observability_provenance_gaps¶

    Observability / Provenance Gaps

    The system lacks the trace detail needed to explain, audit, or reproduce why an action occurred, weakening accountability and recovery.

    Field Value
    Category System Reliability & Observability
    Severity range high -> high

    Typical triggers¶

    • Insufficient structured logging
    • Privacy trade-offs without compensating controls
    • Rapidly evolving tool ecosystems
    • Multi-agent handoffs
    • Nonstandard trace formats

    Detection signals¶

    • Unknown-cause incidents
    • Missing tool arguments or outputs in the trace
    • Low replayability or poor reproducibility
    • Long mean time to resolution for incidents
    • Missing provenance for retrieved content

    Mitigations¶

    • Standardize structured trace schemas
    • Add component IDs and provenance tags
    • Evaluate monitors and postmortems against trace completeness
    • Scale auditability to deployment risk
