    weak_fallback_silent_degradation¶

    The agent silently enters a degraded operating mode such as a fallback model, reduced toolset, cached retrieval path, or heuristic shortcut without surfacing that downgrade to the operator.

    Property Value
    Title Weak Fallback (Silent Degradation)
    Abstract No
    Parent system_reliability_observability

    Attributes¶

    Attribute Range Description
    degraded_mode string Fallback model, tool path, or degraded operating mode observed in the trace.
    fallback_trigger string Immediate trigger associated with the fallback activation.

    Taxonomy Metadata¶

    • Severity range: medium -> critical
    • Typical triggers:
    • Tool or API failures
    • Rate limits or partial outages
    • Token or compute constraints
    • Missing permissions on the preferred path

    • Uninstrumented graceful-degradation logic

    • Detection signals:

    • Sudden changes in model identifiers across adjacent events
    • Tool-call volume or success rate drops
    • Spikes in best-effort or fallback phrasing
    • Divergence between offline evaluation quality and online trace behavior

    • Increased run-to-run variance after a failure

    • Mitigations:

    • Emit explicit degraded-mode flags in telemetry and user-visible output
    • Fail closed for high-stakes operations rather than silently degrading
    • Evaluate fallback paths independently from primary paths

    • Maintain incident playbooks for partial outages and fallback activation

    • Related failure modes:

    • verification_failure
    • observability_provenance_gaps
    • resource_exhaustion_agent_denial_of_service

    choice_overload_premature_commitment¶

    Under a large branching factor, the agent commits too early to one plan, interpretation, or tool without sufficient exploration or backtracking.

    Property Value
    Title Choice Overload (Premature Commitment)
    Abstract No
    Parent agent_cognition_orchestration

    Taxonomy Metadata¶

    • Severity range: medium -> high
    • Typical triggers:
    • Ambiguous goals
    • High-dimensional action spaces
    • Long-horizon tool-rich tasks
    • Budgeted reasoning that forces early stopping

    • Missing explore-versus-exploit logic

    • Detection signals:

    • Low plan diversity in reasoning traces
    • Early tool lock-in despite ambiguous inputs
    • Frequent late replans after initial commitment
    • Repeated failure on tasks with large branching factors

    • Low sensitivity to new evidence after an initial choice

    • Mitigations:

    • Use deliberative search with checkpoints and backtracking
    • Add reject or ask-human branches for uncertain tool selection
    • Allocate an explicit exploration budget before irreversible steps

    • Stress test tasks with multiple plausible solution paths

    • Related failure modes:

    • goal_intent_mismatch

    premature_termination¶

    The agent declares success or stops executing before the objective has been verified or completed.

    Property Value
    Title Premature Termination
    Abstract No
    Parent agent_cognition_orchestration

    Attributes¶

    Attribute Range Description
    validation_surface string Validator, test surface, or explicit completion check expected before termination.

    Taxonomy Metadata¶

    • Severity range: low -> high
    • Typical triggers:
    • Budget constraints
    • Weak termination criteria
    • Tool errors misread as success
    • Missing progress metrics

    • Lack of validator-gated completion

    • Detection signals:

    • Final done-like messages without a preceding verification step
    • Stopping after partial execution despite unresolved work
    • Success claims with no validator or test output
    • Missing state deltas that confirm completion

    • Traces that end immediately after an error without recovery

    • Mitigations:

    • Gate completion on validators, tests, or explicit confirmation
    • Require evidence of completion in the trace
    • Add safe-abort and escalation paths instead of silent early exit

    • Track progress metrics independently of assistant text

    • Related failure modes:

    step_repetition_loop¶

    Property Value
    Title Step Repetition Loop
    Abstract No
    Parent agent_cognition_orchestration

    Attributes¶

    Attribute Range Description
    repeated_tool_signature string Tool signature repeatedly invoked without meaningful state change.
    repeated_call_count integer Number of repeated identical calls observed in the loop.

    Taxonomy Metadata¶

    • Severity range: low -> high
    • Typical triggers:
    • Inadequate loop detection
    • Missing environment state deltas
    • Weak progress metrics
    • Tool failures followed by naive retries

    • Miscalibrated utility or reward heuristics

    • Detection signals:

    • Repeated identical tool calls with the same arguments
    • Cyclic action sequences
    • Unchanged environment outputs across retries
    • Rising cost with no new evidence

    • Repeated attempts after a deterministic schema failure

    • Mitigations:

    • Add loop detectors and repeated-action caps
    • Compare tool results before retrying
    • Escalate after bounded retries instead of continuing blindly

    • Use validator-gated state changes to confirm progress

    • Related failure modes:

    • premature_termination
    • tool_misuse

    tool_misuse¶
